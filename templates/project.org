# PROJECT TEMPLATE v1.0
# This file defines the standard structure for all project notes.
# 
# PLACEHOLDERS (automatically replaced):
#   {{TITLE}}      - Project name from user input
#   {{DATE}}       - Current date (YYYY-MM-DD)
#   {{DATETIME}}   - Current date and time ([YYYY-MM-DD Www HH:MM])
#   {{CATEGORY}}   - Sanitized project name for Org-agenda
#   {{DEADLINE}}   - Default deadline (7 days from now)
#
# CONVENTIONS:
#   - Use TODO/NEXT/INPROGRESS/WAITING/DONE/CANCELLED states
#   - Effort estimates: 0:15, 0:30, 1:00, 2:00, 4:00, 8:00, 16:00
#   - Each phase tagged with :phase-N:
#   - Update kanban: C-c C-c on #+BEGIN line
#   - Update clocktable: C-c C-c on #+BEGIN line
#   - Track time: C-c C-x C-i (clock in), C-c C-x C-o (clock out)
#
# USAGE:
#   Create new project: C-c n p
#   Open project: C-c p o
#   View all projects in agenda: C-c a p
#
# TIPS:
#   - Keep phases focused (3-5 major phases max)
#   - Break large tasks into subtasks with [/] or [%]
#   - Use priorities: [#A] urgent, [#B] important, [#C] nice-to-have
#   - Set DEADLINE for time-sensitive tasks
#   - Set SCHEDULED for planned work
#   - Clock in when starting work (C-c C-x C-i)
#   - Use :PROPERTIES: for metadata (effort, context, etc.)
