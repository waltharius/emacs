* Notes Menus (C-c n)
:PROPERTIES:
:CUSTOM_ID: menu-notes-main
:END:

Main entry point for all note operations. Hierarchical structure:
=c= Create, =f= Find, =i= Insert, =d= Document, =x= Export, =v= View,
=t= Tools, =l= Philosophy (dynamic), =h= Function Help.

Module: =modules/12-transient.el=

** C-c n c — Create / Capture
:PROPERTIES:
:CUSTOM_ID: menu-notes-create
:END:

Create notes and manage captures. Note types in the first column,
capture commands in the second.

*** n — New note
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-base
:END:

Creates a plain Denote note. Prompts for title, tags (space-separated),
and silo (=pks= / =docu= / =journal=).

Function: =my/denote-base= | Module: =modules/05-notes.el=

*** j — Journal today
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-journal
:END:

Opens today's journal. If it exists, appends a new timestamped heading
(=* HH:MM=) at the bottom. If not, creates a new file in =~/notes/journal/=
with front matter and =:well-being:= property.

Function: =my/denote-journal= | Module: =modules/05-notes.el=

*** J — Journal date
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-journal-date
:END:

Creates or opens a journal for an arbitrary date (backdating). Uses
=org-read-date=. Existing journal gets a =* Uzupełnienie= heading with
=:ADDED_AT:= / =:EVENT_DATE:= properties. New backdated files use
=T000000= identifier.

Function: =my/denote-journal-date= | Module: =modules/05-notes.el=

*** e — Essay
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-essay
:END:

Creates note prefixed =ESEJ:= with tags =esej=, =project=, plus a
user-provided project tag. Inserts scaffold: Metadata, Essay Plan,
Bibliography, Working Notes. Always saved in =pks= silo.

Function: =my/denote-essay= | Module: =modules/05-notes.el=

*** L — Linked note
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-linked-note
:END:

Creates a new note linked bidirectionally to the current buffer.
Requires =#+identifier:= in source note. Source gets a forward link,
new note gets a =:BACKLINK:= property. Layout: source left, new note
right in split window.

*Warning:* only works from a Denote =.org= file with =#+identifier:=.

Function: =my/denote-linked-note= | Module: =modules/05-notes.el=

*** i — Ideas capture
:PROPERTIES:
:CUSTOM_ID: fn-my-capture-idea
:END:

Fires capture template =j= directly, no menu. Opens capture buffer to
the right of current window. Records =:SOURCE:= property automatically.

Function: =my/capture-idea= | Module: =modules/06-capture.el=

*** c — Capture menu
:PROPERTIES:
:CUSTOM_ID: fn-org-capture
:END:

Standard =org-capture= menu with all templates. Also global =C-c c=.

Function: =org-capture= (built-in)

*** m — Promote to note
:PROPERTIES:
:CUSTOM_ID: fn-my-capture-promote-to-note
:END:

Promotes heading at point in captures file to a new Denote note.
Preserves =:SOURCE:= as =Source:= line. Removes original heading after
success.

*Warning:* destructive — deletes the original heading, no automatic undo.
Commit captures.org before running on important entries.

Function: =my/capture-promote-to-note= | Module: =modules/06-capture.el=

*** t — Transclusion submenu
:PROPERTIES:
:CUSTOM_ID: menu-notes-transclusion
:END:

Obsidian-style transclusion for Denote notes: pick a note by title
search, then pick whole note or a heading. Generates/reuses CUSTOM_ID
for headings automatically.

Module: =modules/20-transclusion.el=

**** a — Add transclusion (wizard)
:PROPERTIES:
:CUSTOM_ID: fn-my-denote-transclude-insert
:END:

Prompts for a note (Denote completion), then whole note or heading.
Generates CUSTOM_ID from heading title (spaces → hyphens) if missing,
saves it into the SOURCE file. Inserts a =#+transclude:= /
=#+INCLUDE:= pair at point and renders it immediately.

*Warning:* writes to the source file's PROPERTIES drawer if the
heading had no CUSTOM_ID yet. Commit before heavy first use.

Function: =my/denote-transclude-insert=

**** A — Add all in buffer
:PROPERTIES:
:CUSTOM_ID: fn-org-transclusion-add-all
:END:

Activates every =#+transclude:= keyword in the current buffer at once.

Function: =org-transclusion-add-all= (org-transclusion package)

**** g — Refresh at point
:PROPERTIES:
:CUSTOM_ID: fn-org-transclusion-refresh
:END:

Re-reads the source and updates the transcluded copy at point after
the source has changed.

Function: =org-transclusion-refresh= (org-transclusion package)

**** r — Remove at point
:PROPERTIES:
:CUSTOM_ID: fn-org-transclusion-remove
:END:

Removes the transcluded content at point, restoring the plain
=#+transclude:= keyword line. Non-destructive to the source file.

Function: =org-transclusion-remove= (org-transclusion package)

**** T — Toggle mode
:PROPERTIES:
:CUSTOM_ID: fn-org-transclusion-mode
:END:

Toggles =org-transclusion-mode= for the buffer. With
=org-transclusion-add-all-on-activate= set to =nil= (our config), it
does NOT auto-transclude everything on activation.

Function: =org-transclusion-mode= (org-transclusion package)

**** m — Full org-transclusion menu
:PROPERTIES:
:CUSTOM_ID: fn-org-transclusion-transient-menu

Function: =org-transclusion-transient-menu= (org-transclusion package)

working tree. Commit or stash important changes first.

** C-c n x — Export
:PROPERTIES:
:CUSTOM_ID: menu-notes-export
:END:

Export notes to PDF using LuaLaTeX. Output mirrors silo structure in
=~/notes/pdf/=. Single-file export also available as global =C-c p=.
Module: =modules/16-org-export.el=

*** p — Export to PDF
:PROPERTIES:
:CUSTOM_ID: fn-my-org-export-to-pdf
:END:

Exports current buffer to PDF. Language auto-detected from
=#+filetags:=: =:en:= → English, default → Polish. Journal files
(=:journal:=) use Playpen Sans Hebrew font. Filename from =#+title:=.
Prompts on overwrite: =o= overwrite, =r= rename with index, =q= cancel.
Build files isolated in =/tmp=, deleted after export.

Function: =my/org-export-to-pdf= | Module: =modules/16-org-export.el=

*** P — Batch PDF — ANY keyword
:PROPERTIES:
:CUSTOM_ID: fn-my-org-export-pdf-by-keyword
:END:

Exports all notes whose filenames match *any* of the given keywords.
Keyword input via =completing-read-multiple= with TAB completion over
all Denote tags in use. Results shown in =*PDF Export Results*= buffer.

Function: =my/org-export-pdf-by-keyword= | Module: =modules/16-org-export.el=

*** Q — Batch PDF — ALL keywords
:PROPERTIES:
:CUSTOM_ID: fn-my-org-export-pdf-by-all-keywords
:END:

Exports all notes whose filenames match *all* given keywords
simultaneously. More selective than =P=; useful for scoped sets
(e.g. =esej= + =project=).

Function: =my/org-export-pdf-by-all-keywords= | Module: =modules/16-org-export.el=

* Capture System
:PROPERTIES:
:CUSTOM_ID: capture-system
:END:

Captures land in the file defined by =my-journal-captures= (variable
in =00-core.el=). Template =j= creates an entry under =* Ideas= with a
=:SOURCE:= property linking back to the originating note.

The right-side window behaviour is implemented via
=org-capture-mode-hook= calling =my/capture--show-right=, which splits
the origin window and places the capture buffer on the right. This
only fires when capture is started via =my/capture-idea=; generic
=org-capture= is unaffected.

Module: =modules/06-capture.el=
